# ID 2.2 - Prototype Implementation Plan

This document explains how to build CampusCast AI step by step.

## Final Workflow

```text
Manual Trigger
    -> HTTP Request: Open-Meteo
    -> Read or define campus events
    -> Code or Set Node: prepare data
    -> HTTP Request: Ollama
    -> HTTP Request: Kokoro
    -> Google Sheets or SQLite: save result
```

## Step 1 - Create the Trigger

Tool:

- n8n

What to do:

- create a new workflow
- add a Manual Trigger node

Why:

- this lets you run the prototype manually during testing

Expected result:

- clicking "Execute Workflow" starts the automation

Later improvement:

- replace Manual Trigger with Schedule Trigger to run daily

## Step 2 - Collect Weather Data

Tool:

- n8n HTTP Request node

Configuration:

```text
Method: GET
URL: https://api.open-meteo.com/v1/forecast?latitude=-25.4284&longitude=-49.2733&current=temperature_2m,relative_humidity_2m,precipitation,rain,weather_code,wind_speed_10m&timezone=America%2FSao_Paulo
```

Why:

- this is the main public API input

Expected result:

- JSON data with current weather values

## Step 3 - Add Campus Events

Beginner option:

- create event data in a Google Sheet

Alternative for the first prototype:

- use a Set node in n8n with two or three sample events

Example events:

```text
2026-04-23,19:00,AI Study Group,Lab 3,Computer Science students,medium
2026-04-23,21:00,Project Deadline Reminder,Online,All students,high
```

Why:

- this makes the project more creative than a simple weather automation
- the LLM can generate a richer campus bulletin

Expected result:

- the workflow has weather data and event data before calling the LLM

## Step 4 - Prepare the Data

Tool:

- n8n Set node or Code node

Fields to prepare:

```text
city
weather_time
temperature
humidity
rain
precipitation
wind_speed
events_list
```

Why:

- clean data makes the LLM prompt more reliable

Expected result:

- one structured object that can be inserted into the prompt

## Step 5 - Generate the Bulletin with Ollama

Tool:

- n8n HTTP Request node

URL:

```text
http://localhost:11434/api/generate
```

Method:

```text
POST
```

Body type:

```text
JSON
```

Example body:

```json
{
  "model": "llama3.2",
  "prompt": "You are CampusCast AI. Create a short Portuguese campus audio bulletin using this data: City: Curitiba. Temperature: {{$json.temperature}} C. Humidity: {{$json.humidity}}%. Rain: {{$json.rain}} mm. Wind speed: {{$json.wind_speed}} km/h. Events: {{$json.events_list}}. Maximum 120 words. Do not invent events. Make it suitable for text-to-speech.",
  "stream": false
}
```

Why:

- Ollama transforms structured data into a useful human-readable bulletin

Expected result:

- generated text in the response field

## Step 6 - Generate Audio with Kokoro

Tool:

- n8n HTTP Request node or local command wrapper

What to send:

- the text generated by Ollama

Why:

- Kokoro converts the bulletin into audio, improving accessibility

Expected result:

- generated audio file such as `campuscast_2026-04-22.mp3` or `.wav`

Implementation note:

- Kokoro setups vary. If your Kokoro runs as an API server, call it directly from n8n.
- If Kokoro only runs as a local script, create a small local endpoint or run it manually for the first prototype and document the step.

## Step 7 - Save the Result

Tool:

- Google Sheets node in n8n
- or SQLite node / local database

Save these fields:

```text
timestamp
city
temperature
humidity
rain
precipitation
wind_speed
events_used
llm_response
audio_file
status
error_message
```

Why:

- this proves the output was generated and recorded

Expected result:

- one new row for each workflow execution
